ejemploDoc
==========

.. automodule:: ejemploDoc
   :members:
   :undoc-members:
   :show-inheritance:


.. arriba se pone el nombre del módulo a documentar, los '=' indican un título de nivel superior
.. automodule -> le dice a sphinx que debe buscar este archivo 'EjemploDoc' en los documentos y hacer su documentación
.. members -> indica que se debe incluir la documentación de todas las clases, funciones y variables que estén en el módulo
.. undoc-members -> incluye lo que no es docstrings (por si quieres asegurar que todo el código se documente)
.. show-inheritance -> muestra la relación de herencias de clases, sirve para entender más las jerarquías
